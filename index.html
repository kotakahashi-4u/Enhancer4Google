プライバシーポリシー

「Enhancer 4 Google」（以下、本拡張機能）は、ユーザーのプライバシーを最優先に考えて設計されています。本ポリシーでは、本拡張機能がどのようにデータを扱い、保護するかについて説明します。

要約: 本拡張機能は、ユーザーの個人情報、閲覧履歴、入力データを開発者のサーバーへ送信・収集することは一切ありません。すべての処理はユーザーのブラウザおよびGoogleの公式APIとの直接通信によって行われます。

データの収集と使用について

1. データの保存場所

本拡張機能の設定（ON/OFF状態やパラメータ）は、すべてお客様のブラウザ内（Chrome Storage）に保存されます。これらが外部サーバーへ送信されることはありません。

2. 外部通信とデータ送信

本拡張機能は、以下の特定の機能を利用する場合にのみ、外部（Google API または指定されたウェブサイト）と通信を行います。

  • Google Slides 連携機能: PDFデータをスライドに変換するため、Google Drive API および Google Slides API を使用します。PDFから変換された画像データは一時的にお客様自身のGoogleドライブにアップロードされ、スライドへの挿入完了後に即座に削除されます。このプロセスにおいて、データが開発者や第三者のサーバーへ送信されることはありません。
  • Webクローラー機能: お客様が指定したURLの情報を取得するために、そのウェブサイトへアクセスします。取得したデータはブラウザ内でMarkdownファイルに変換され、ダウンロードされます。開発者のサーバーへ送信されることはありません。

権限の使用目的

本拡張機能は、機能の実現に必要な最小限の権限のみを使用します。

  • identity: GoogleドライブおよびGoogleスライドを操作するためのOAuth2認証トークンを取得するために使用します。
  • drive.file: スライド作成時に使用する一時画像ファイルを、Googleドライブへアップロード・削除するために使用します。本拡張機能が作成したファイルのみにアクセスし、お客様の既存のファイルには一切アクセスしません。
  • presentations: Googleスライドを新規作成し、編集するために使用します。
  • storage: ユーザーの設定情報をブラウザ内に保存するために使用します。
  • alarms: NotebookLMの同期スケジューラー（定期実行）機能のために使用します。
  • host_permissions (<all_urls>): 「Webクローラー機能」において、ユーザーが指定した任意のウェブサイトの情報を取得・解析し、NotebookLM用のソースを作成するために必要です。この権限は、ユーザーが明示的にクローリングを実行したときのみ行使されます。

Google API ユーザーデータの取り扱い

本拡張機能は、Google APIから取得した情報の使用および他のアプリへの転送において、Google API Services User Data Policy（Limited Use requirementsを含む）を遵守します。

  • 本拡張機能は、スライド作成機能を実行する目的でのみGoogleユーザーデータ（Driveファイル、スライド）を使用します。
  • ユーザーデータが人間の目に触れることはなく、広告目的で使用されることもありません。
